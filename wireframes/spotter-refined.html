The Professional Rule

  • If the user thinks "cool animation" → too much. If they subconsciously feel "this app is working" without noticing the particles → perfect.
  • Google Lens uses ~3 particles at a time. Apple Live Text uses just the sweep line. Samsung Bixby uses corner brackets only. We combined all three at their most subtle levels.
  • One particle type eliminates visual noise. The star reads as "sparkle" at any size without needing variety for interest.

Adaptive Density = Subconscious Guidance

  • Active (3-5 stars) — "The app is analyzing." User keeps panning naturally.
  • Idle (1-2 stars, no sweep) — "It's slowing down… I should move." The nudge text is backup; the particle thinning communicates first.
  • New area burst (6-8, then settle) — "It noticed I moved." Reinforces that the AI is responsive without being flashy.

Performance

  • Max 5 DOM elements at any time (down from 20). Each is a 2×2px div with opacity + transform animation.
  • GPU-composited — zero layout, zero paint. Unmeasurable overhead.
  • No canvas involvement. Particles are HTML overlay on native <video> element.
