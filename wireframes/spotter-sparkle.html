Sparkle System Breakdown

Three Particle Types

  • Star (4-point cross) — white/green, 3-5px, rotates slightly as it fades. Primary particle.
  • Diagonal star — rotated 45°, smaller, adds variety so it doesn't feel repetitive.
  • Soft glow dot — radial gradient, 6px, blurred. Ambient filler between stars.

Adaptive Density (3 States)

  • Active scanning — 15-20 particles visible at once. New ones spawn every 150-250ms. Sweep line active.
  • Lingering / idle — 3-5 particles, spawn every 600-800ms. Sweep line stops. Communicates "I'm done here."
  • New area burst — 25-30 particles in a 0.5s burst, then settles to active rate. Feels like the AI "woke up."

Performance Cost

  • Pure CSS animations — zero canvas work, zero JS per frame.
  • Each particle is a tiny DOM element with a CSS animation + opacity/transform.
  • GPU-composited (opacity + transform only) — no layout or paint thrashing.
  • Spawn/despawn via JS setInterval — particles self-remove after animation ends.
  • Total overhead: <1ms/frame on any device. Negligible.

Why This Works

  • Fills the "empty camera" gap — without sparkles, Spotter Mode looks like a plain camera app. Users wonder "is it doing anything?"
  • Google Lens precedent — users already associate this particle style with "AI is analyzing."
  • Adaptive density = status communication — busy sparkles = "I'm finding things." Calm = "I'm done, move on." No text needed.
  • Zero inference overhead — purely cosmetic CSS. Doesn't touch the detection pipeline at all.
