UX Strategy Comparison

Performance & experience analysis for BundleBox real-time scanning

⭐ Recommendation: Spotter Mode (Option A)

Spotter Mode is the right choice because your core problem is inference latency, not visual fidelity. The user doesn't need to see where each object is — they need to know the AI saw it. Toasts communicate acknowledgment instantly. Pulse Pins look fancier but re-introduce per-frame rendering overhead, bbox dependency, and visual clutter — the same problems you're solving.

Why Spotter Mode Wins

Your stated goal: "We want to tell the user I see this object, that's it."

Spotter Mode is a direct translation of that requirement into UX:

  • Eliminates all rendering overhead — no canvas, no frame store, no synced mode. Video plays at native speed.
  • Faster perceived performance — user sees smooth 30fps camera feed instead of choppy synced frames. The app feels instant even though inference speed is unchanged.
  • Toasts work during motion — bbox pins jitter and lag during camera pan. Text notifications don't. Your users will be panning constantly.
  • Guides the user — the "move on" nudge creates a natural scanning cadence. Users finish faster with better coverage.
  • Client-side throttling becomes trivial — since there's no visual sync needed, you can freely skip frames when the scene is static. 3-5x fewer inference calls.
  • Simpler code, fewer bugs — removes frame store, velocity interpolation, mask rendering, synced mode compositing. ~200 lines of JS deleted.

Pulse Pin is the better choice only if spatial AR is a core selling point for investors. For actual user utility, Spotter Mode is superior.
